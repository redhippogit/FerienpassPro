01 — Ausgangslage

Das Problem ist real.
Die Lösung auch.

Warum jetzt

Die aktuelle Ferienpass-Plattform lässt alle drei Nutzergruppen im Stich: Eltern kämpfen mit fragmentierten Informationen und fehlendem Online-Booking, Anbieter verwalten alles manuell, das Ferienpass-Team hat keinen zentralen Überblick. goSNOB.ai liefert die Infrastruktur, die alle drei verdienen — dreisprachig, KI-gestützt, in 15 Wochen.

Eltern

Fragmentiert & chaotisch

Dutzende Websites, kein Kalender, keine Reviews, kein TWINT. Buchung via E-Mail im Jahr 2026.

Kursanbieter

Alles manuell

Kein Self-Service-Portal. Jede Änderung = Telefonat. Keine Echtzeit-Kapazitätsanzeige.

Ferienpass-Team

Kein Überblick

Kein Dashboard, keine automatisierte Kommunikation, kein skalierbares Approval-System.

06 — Rahmenbedingungen

Was wir voraussetzen.
Was der Kunde liefert.

⚠

Kritische Voraussetzungen — müssen in Woche 1–2 geklärt sein

Verzögerungen bei Stripe-Zugang oder Hosting-Entscheid verschieben den Go-Live-Termin direkt. Diese Punkte sind nicht optional.

Stripe-Zugang (kritisch)

Zugang zum Stripe-Account (Test + Prod) ab Projekt-Start erforderlich. Ohne Stripe-Zugang kann kein Zahlungsflow gebaut werden.

Hosting-Entscheid Wo 1–2 (kritisch)

Betrifft gesamte Architektur: Datenbank, Deployment, File Storage. Empfehlung: Managed Services (Vercel / Supabase). Spart Zeit, reduziert Portabilität.

Drittanbieter-Konten (kritisch)

Kunde erstellt/besitzt Konten für E-Mail-Delivery (SendGrid/Mailgun), Image-Hosting etc. und gibt goSNOB.ai Zugang. Setup durch uns = zusätzlicher Aufwand.

Zahlungsmodell

Ferienpass zentralisiert alle Familienzahlungen. Anbieter-Auszahlungen sind separater Business-Prozess — kein Platform-Feature in Phase 1. Marketplace Payouts = Phase 2.

Kein Datenmigration

Greenfield-Plattform — keine Migration aus bestehendem System. Anbieter registrieren sich selbst. Initiale Inhalte werden durch Ferienpass-Team via Admin-Modul befüllt.

Deutschen Copy vom Kunden

Statische Seiteninhalte (Homepage, About, FAQ, Hilfe-Texte) werden vom Kunden auf Deutsch geliefert. goSNOB.ai implementiert und adaptiert für EN/FR.

Kursinhalt-Sprache

Provider-eingereichte Kursbeschreibungen erscheinen in Originalsprache. Platform-UI und System-E-Mails sind dreisprachig. Automatische Übersetzung = Phase 2.

Subventions-Validierung

Dieses Proposal geht davon aus, dass Rabatte und Ferienpass-Karten offline abgewickelt werden. Digitale Validierung beim Checkout = zusätzlicher Scope, vor Vertragsabschluss zu definieren.

Hosting-Kosten (Post-Launch)

Laufende Cloud-Kosten (Hosting, Stripe-Fees, E-Mail-Delivery) gehen auf den Kunden. Geschätzt: unter CHF 200/Monat bei Launch-Scale. Exkl. Stripe-Transaktionsgebühren.

Ein Staging-Environment

Ein Environment, das zum Go-Live zu Production wird. Kein separates Dev/Staging/Prod-Drei-Environment-Setup im Scope.

Registrierungspflicht

Jede Aktion über das Browsen hinaus (inkl. Ferienpass-Kauf) erfordert Registrierung und Login. Kein Guest Checkout im Scope.

CI/CD via GitHub Actions

Continuous Integration und Deployment via GitHub Actions mit platform-nativem Deployment. Kein Custom-Infrastructure-Management im Scope.

Häufige Fragen

Kann ich Add-ons nach Vertragsabschluss noch hinzufügen?

+
Add-ons müssen grundsätzlich vor Vertragsabschluss definiert werden, da sie Scope und Timeline beeinflussen. Einige technisch unkomplizierte Add-ons (z.B. XLSX-Export) können auch später separat beauftragt werden — in Absprache und gegen Aufpreis.

Was passiert, wenn wir keinen Stripe-Account haben?

+
Ein Stripe-Account muss vor Projekt-Start eingerichtet und goSNOB.ai zugänglich gemacht werden. Wir können bei der Einrichtung begleiten, aber der Account liegt beim Kunden. Ohne Stripe-Zugang kann der gesamte Zahlungsflow nicht gebaut werden.

Wer schreibt die Inhalte (Texte, FAQs, About)?

+
Der Kunde liefert alle statischen Seiteninhalte auf Deutsch. goSNOB.ai implementiert und adaptiert für EN/FR. Originalinhalte werden nicht verfasst — Content-Erstellung ist kein Teil des Scopes.

Ist die Plattform nach Go-Live wartungsfrei?

+
Das Ferienpass-Team übernimmt nach Go-Live den operativen Betrieb via Admin-Modul. Laufende technische Wartung (Updates, Sicherheits-Patches, Bug-Fixes) ist separat zu beauftragen. Wir empfehlen einen Wartungsvertrag für Phase 2.

Wie wird DSGVO / DSG (Schweiz) berücksichtigt?

+
Die Plattform wird DSGVO-konform gebaut: Datenminimierung, Consent-Management, sichere Verarbeitung. Daten Minderjähriger werden besonders geschützt. Kommunikationsfeatures (Phase 2) werden mit besonderer DSGVO-Prüfung umgesetzt. Rechtliche Beratung liegt beim Kunden.

Können Anbieter Kursinhalte auf EN/FR übersetzen?

+
Phase 1: Kursinhalte erscheinen in der Originalsprache des Anbieters. Automatische KI-Übersetzung von Kursinhalten ist Phase 2. Anbieter können optional mehrere Sprachversionen manuell einpflegen, wenn das Formular-Setup dies erlaubt.
