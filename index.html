Delivery Plan

15 Wochen.
Go-Live Sommer 2026.

Kritische Voraussetzungen

Stripe-Zugangsdaten (Test + Prod) ab Kick-off. Hosting-Entscheid in Woche 1–2 (Vercel / Supabase empfohlen). Beides blockiert den Start.

Wo 1–2
Kick-off & Alignment
Technisches Setup. Design-Brief bestaetigt. Hosting-Entscheid. Stripe-Zugang konfiguriert. Offene Fragen abgeschlossen.
Wo 2–4
UX & Design
15–20+ Screens designt und freigegeben: Homepage, Kurslistung, Buchungsflow, Anbieter-Portal, Admin-Modul. 2–3 Iterationsrunden.
Wo 4–9
Build — Oeffentliche Plattform
Dreisprachige Website live auf Staging: Homepage, Kurslistung, Detailseiten, Familienkonten, Buchungs- und Zahlungsflow.
Wo 9–12
Build — Portale & Integrationen
Anbieter-Portal, Admin-Modul, E-Mail-Automatisierung (DE/EN/FR), goSNOB.ai KI-Chatbot — alle auf Staging fertig.
Wo 12–14
Testing & UAT
End-to-End-Testing. Client-Team fuehrt User Acceptance Testing auf Staging durch. Bugs behoben, Feedback eingearbeitet.
Wo 15
Go-Live
Production Deployment. Plattform live. Ferienpass-Team uebernimmt via Admin-Modul. Handover-Dokumentation uebergeben.

Rahmenbedingungen

Was wir voraussetzen.

Kritisch — vor Woche 1–2 klaeren

Verzoegerungen bei Stripe-Zugang oder Hosting verschieben den Go-Live direkt. Nicht optional.

Stripe-Zugang

Test + Prod Credentials ab Projekt-Start. Ohne Stripe kein Zahlungsflow moeglich.

Hosting-Entscheid Woche 1–2

Betrifft Architektur, DB, Deployment, File Storage. Empfehlung: Vercel / Supabase.

Drittanbieter-Konten

E-Mail-Delivery (SendGrid/Mailgun) — Kunde erstellt, goSNOB.ai erhaelt Zugang. Gilt auch fuer Image-Hosting.

Zahlungsmodell

Ferienpass zentralisiert alle Zahlungen. Anbieter-Auszahlungen offline. Marketplace Payouts = Phase 2.

Kein Datenmigration

Greenfield-Plattform. Anbieter registrieren selbst. Initiale Inhalte via Admin-Modul befuellt.

Copy vom Kunden

Statische Inhalte (Homepage, About, FAQ) werden auf Deutsch geliefert. goSNOB.ai adaptiert fuer EN/FR.

Kursinhalt-Sprache

Kursbeschreibungen in Originalsprache des Anbieters. Platform-UI dreisprachig. Auto-Uebersetzung = Phase 2.

Subventions-Validierung

Rabatte und Karten-Rabatte offline abgewickelt. Digitale Validierung beim Checkout = separater Scope.

Laufende Kosten

Hosting, E-Mail-Delivery, Stripe-Transaktionsgebuehren gehen auf den Kunden. Unter CHF 200/Monat erwartet.

Ein Environment

Staging wird zu Production. Kein Dev/Staging/Prod-Drei-Environment-Setup im Scope.

Registrierungspflicht

Jede Aktion ueber Browsen erfordert Login. Kein Guest Checkout im Scope.

CI/CD via GitHub Actions

Deployment via GitHub Actions mit platform-nativem Deployment. Kein Custom-Infrastructure-Management.

FAQ

Kann ich Add-ons nach Vertragsabschluss noch hinzufuegen?

+
Grundsaetzlich muessen Add-ons vor Vertragsabschluss definiert werden, da sie Scope und Timeline beeinflussen. Technisch einfache Add-ons wie XLSX-Export koennen spaeter separat beauftragt werden.

Was passiert ohne Stripe-Account?

+
Ein Stripe-Account muss vor Projekt-Start eingerichtet und goSNOB.ai zugaenglich gemacht werden. Ohne Stripe-Zugang kann der gesamte Zahlungsflow nicht entwickelt werden.

Wer schreibt die Inhalte (Texte, FAQs)?

+
Der Kunde liefert alle statischen Seiteninhalte auf Deutsch. goSNOB.ai implementiert und adaptiert fuer EN/FR. Content-Erstellung ist kein Teil des Scopes.

DSGVO / DSG (Schweiz)?

+
Die Plattform wird DSGVO-konform gebaut: Datenminimierung, Consent-Management, sichere Verarbeitung. Daten Minderjaehriger werden besonders geschuetzt. Rechtliche Beratung liegt beim Kunden.

Wie kontaktiere ich goSNOB.ai?

+
Alle Anfragen direkt an team@gosnob.com — kein Pitch, nur SNOB.
