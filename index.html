01 — Ausgangslage

```

Warum jetzt

Die aktuelle Ferienpass-Plattform lässt alle drei Nutzergruppen im Stich: Eltern kämpfen mit fragmentierten Informationen und fehlendem Online-Booking, Anbieter verwalten alles manuell, das Ferienpass-Team hat keinen zentralen Überblick. goSNOB.ai liefert die Infrastruktur in 15 Wochen — dreisprachig, KI-gestützt.

Eltern

Fragmentiert & chaotisch

Dutzende Websites, kein Kalender, keine Reviews, kein TWINT. Buchung via E-Mail im Jahr 2026.

Kursanbieter

Alles manuell

Kein Self-Service-Portal. Jede Änderung = Telefonat. Keine Echtzeit-Kapazitätsanzeige.

Ferienpass-Team

Kein Überblick

Kein Dashboard, keine automatisierte Kommunikation, kein skalierbares Approval-System.

Unser Ansatz

Zwei Phasen. Eine Vision.

Phase 1 — Base Package

Mai → August 2026

  • Dreisprachige Website (DE/EN/FR)
  • Kurssuche, Filter & Detailseiten
  • Familienkonten & Online-Buchungsflow
  • Stripe-Zahlung (TWINT, Apple/Google Pay)
  • Anbieter Self-Service Portal
  • Ferienpass Admin- & Approval-Modul
  • Automatisierte E-Mails (dreisprachig)
  • goSNOB.ai KI-Chatbot Integration
  • UX Design (15–20+ Screens)
Phase 2 — Further Development

Ab Herbst 2026

  • Kids-Profile & altersbasiertes AI-Matching
  • Mobile App / PWA mit Push-Notifikationen
  • SMS / WhatsApp Kanal
  • Vollständiges Wartelisten-Management
  • Reviews & Ratings (full)
  • Analytics Dashboards
  • Automatische Kursübersetzung (KI)
  • Stripe Split Payments / Marketplace Payouts
```

06 — Rahmenbedingungen

Was wir voraussetzen.
Was der Kunde liefert.

```

Kritische Punkte — müssen in Woche 1–2 geklärt sein

Verzögerungen bei Stripe-Zugang oder Hosting-Entscheid verschieben den Go-Live-Termin direkt. Nicht optional.

Stripe-Zugang

Zugang zum Stripe-Account (Test + Prod) ab Projekt-Start erforderlich. Ohne Stripe kein Zahlungsflow.

Hosting-Entscheid Wo 1–2

Betrifft gesamte Architektur: DB, Deployment, File Storage. Empfehlung: Vercel / Supabase (managed).

Drittanbieter-Konten

Kunde erstellt Konten für E-Mail-Delivery (SendGrid/Mailgun), Image-Hosting und gibt goSNOB.ai Zugang.

Zahlungsmodell

Ferienpass zentralisiert alle Zahlungen. Anbieter-Auszahlungen = separater Prozess, offline. Marketplace Payouts = Phase 2.

Kein Datenmigration

Greenfield-Plattform. Anbieter registrieren sich selbst. Initiale Inhalte via Admin-Modul befüllt.

Deutschen Copy vom Kunden

Statische Inhalte (Homepage, About, FAQ) werden vom Kunden auf DE geliefert. goSNOB.ai adaptiert für EN/FR.

Kursinhalt-Sprache

Kursbeschreibungen in Originalsprache des Anbieters. Platform-UI und System-E-Mails dreisprachig. Auto-Übersetzung = Phase 2.

Subventions-Validierung

Rabatte und Ferienpass-Karten werden offline abgewickelt. Digitale Validierung beim Checkout = separater Scope.

Hosting-Kosten Post-Launch

Cloud-Kosten (Hosting, E-Mail, Storage) gehen auf den Kunden. Geschätzt: unter CHF 200/Monat bei Launch-Scale.

Ein Staging-Environment

Kein separates Dev/Staging/Prod-Setup. Ein Environment, das zum Go-Live zu Production wird.

Registrierungspflicht

Jede Aktion über Browsen hinaus erfordert Login. Kein Guest Checkout im Scope.

CI/CD via GitHub Actions

Deployment via GitHub Actions. Kein Custom-Infrastructure-Management im Scope.

```

Häufige Fragen

Kann ich Add-ons nach Vertragsabschluss noch hinzufügen?

+
Add-ons müssen grundsätzlich vor Vertragsabschluss definiert werden, da sie Scope und Timeline beeinflussen. Einige (z.B. XLSX-Export) können auch später separat beauftragt werden — in Absprache und gegen Aufpreis.

Was passiert, wenn wir keinen Stripe-Account haben?

+
Ein Stripe-Account muss vor Projekt-Start eingerichtet und goSNOB.ai zugänglich gemacht werden. Ohne Stripe-Zugang kann der Zahlungsflow nicht gebaut werden. Einrichtungsbegleitung möglich, geht aber auf Zeitkonto.

Wer schreibt die Inhalte (Texte, FAQs, About)?

+
Der Kunde liefert alle statischen Seiteninhalte auf Deutsch. goSNOB.ai implementiert und adaptiert für EN/FR. Originalinhalte verfassen ist kein Teil des Scopes.

Ist die Plattform nach Go-Live wartungsfrei?

+
Das Ferienpass-Team übernimmt den operativen Betrieb via Admin-Modul. Laufende technische Wartung (Updates, Security-Patches, Bug-Fixes) ist separat zu beauftragen. Wartungsvertrag für Phase 2 empfohlen.

Wie wird DSGVO / DSG (Schweiz) berücksichtigt?

+
Plattform wird DSGVO-konform gebaut: Datenminimierung, Consent-Management, sichere Verarbeitung. Daten Minderjähriger besonders geschützt. Rechtliche Beratung liegt beim Kunden.
